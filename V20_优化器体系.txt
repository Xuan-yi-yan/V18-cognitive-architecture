═══════════════════════════════════════════════════════════════
一、为什么不用一个全局优化器
═══════════════════════════════════════════════════════════════

  V20是8个异构模块的串行链路，各模块学习任务完全不同:
    P7: 学"6166字中哪个跟当前A句相关" — 检索任务
    P6: 学"哪些字序列对应这个sent_vec" — 生成任务
    P3-L: 学"哪些属性组合经常一起出现" — 关联任务
    ABC: 学"P3规则判断是什么样" — 模仿任务
    Gate: 学"loss信号对应开哪些维度" — 调控任务
    Master: 学"各模块损失应该各占多少比例" — 调度任务

  一个全局优化器(lr=0.003)会:
    - P6刚好时 P7已经冲过头
    - Gate应该激进但被P6拖慢
    - Master应该稳健但被P7带飞
    - P3-L精细度被粗粒度lr淹没

═══════════════════════════════════════════════════════════════
二、7个独立Adam优化器
═══════════════════════════════════════════════════════════════

  ┌──────────────┬──────────┬───────────┬──────────────────────┐
  │ 优化器       │ 模块     │ 参数      │ lr倍数(基准0.003)    │
  ├──────────────┼──────────┼───────────┼──────────────────────┤
  │ opt_gate     │ 主Gate   │ 143K      │ ×2.0 (0.006)        │
  │ opt_p7       │ P7路由   │ 226K      │ ×1.5 (0.0045)       │
  │ opt_abc      │ ABC三阶  │ 65K       │ ×1.2 (0.0036)       │
  │ opt_p6       │ P6解码   │ 4.37M     │ ×1.0 (0.003)        │
  │ opt_p3l      │ P3-L联动 │ 158K      │ ×0.8 (0.0024)       │
  │ opt_p7_in    │ p7_input │ 25K       │ ×1.0 (0.003)        │
  │ opt_master   │ Master   │ ~400      │ ×0.5 (0.0015)       │
  └──────────────┴──────────┴───────────┴──────────────────────┘

  各优化器独立管理梯度, 互不干扰。同一个step()调用7个.step()。

  设计逻辑:
    - Gate(×2.0): 必须最快响应loss变化, 慢了P6 encoder死锁
    - P7(×1.5): 全词表6166字检索最耗梯度, 需要略激进
    - ABC(×1.2): 三阶段串行需要协调, 稍快但不能过快
    - P6(×1.0): 4.37M参数最大模块, 标准lr最稳
    - P3-L(×0.8): 312头细粒度, lr高会振荡——每头独立方向被打乱
    - p7_input(×1.0): 投影层, 标准lr
    - Master(×0.5): 温控器, 应该比被控模块慢——否则"治疗比病还重"


═══════════════════════════════════════════════════════════════
三、为什么不是SGD/AdamW/RMSprop
═══════════════════════════════════════════════════════════════

  SGD: 无动量→Gate/P7训练初期振荡。591个loss的全地形下SGD太慢。
  AdamW: 权重衰减对V20有害——P3规则引擎0参数, 不需要正则化。
  RMSprop: 没一阶动量→Gate的快速响应能力不足。

  Adam: 一阶+二阶动量, 自适应调节每维lr。
  适合V20的多loss多模块地形——各维度学习速度不同时自动调。


═══════════════════════════════════════════════════════════════
四、梯度流动路径
═══════════════════════════════════════════════════════════════

  batch_loss.backward() →
    ├── ce_loss → P6(128头) → char_embed(tied权重)
    ├── loss_A → ABC_StageA(15类) → p7_input_proj → P3属性(无梯度)
    ├── loss_B → ABC_StageB(64D) → P3-L(312头) → p7_input_proj
    ├── loss_C → ABC_StageC(48D) → ABC_StageB → ...
    ├── gate_loss → P7(explore_net+meta) → word_gate/sent_gate
    └── Master Gate → 无直接梯度(通过temps间接调控)

  关键梯度流:
    P3-L梯度: attn_scores(mean/std/range全量) → ABC_B → loss_B → backward
    P7层级调制: _loss_vec → explore_net → meta → word_gate/sent_gate
    p7_input_proj: P3属性拼char_emb → 192D→128D → P7 attention → 全程梯度


═══════════════════════════════════════════════════════════════
五、优化器状态与显存
═══════════════════════════════════════════════════════════════

  Adam状态: 每个参数存储m(一阶矩)+v(二阶矩)
  = 参数量的2倍额外显存
  5.27M参数 × 3(参数+m+v) × 4字节 = 63MB 优化器状态

  总计显存:
    模型参数: 5.27M × 4B = 21MB
    优化器状态:         63MB
    训练数据+缓存:     ~2GB
    激活值+计算图:     ~2GB
    ─────────────────────
    训练峰值:          ~5GB (12GB总显存留7GB)


═══════════════════════════════════════════════════════════════
六、为什么这个体系是V20架构的必然产物
═══════════════════════════════════════════════════════════════

  不是"调参调出来的"——是架构需求逼出来的:

  1. 8模块异构 → 需要独立lr (一个lr管所有→互相干扰)
  2. 591独立loss → 需要Adam自适应 (SGD在复杂loss地形上震荡)
  3. 5层Gate分级 → Master必须比子模块慢 (否则温控振荡)
  4. P3-L 312头 → 需要低lr (头越多越敏感)
  5. P7全词表检索 → 需要高lr (检索任务比生成任务更难收敛)

  这7个优化器的lr设定是39天训练迭代的收敛结果,
  不是一次性猜测。每次OOM/NaN/不收敛都修正了对应模块的lr。
